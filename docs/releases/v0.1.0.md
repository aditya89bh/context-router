# context-router v0.1.0

## Summary

`context-router` v0.1.0 is a lightweight Python developer tool for routing the right context to the right agent at the right time. It demonstrates a production-oriented separation between memory storage, routing, token budgeting, packaging, evaluation, and observability.

## Core features

- `ContextItem`, `ScoredContextItem`, and `ContextPack` data structures
- In-memory `MemoryStore`
- Standard-library `SQLiteMemoryStore` with context manager support and FTS-backed search when available
- `BaseRouter` contract
- `RecencyRouter`, `SemanticRouter`, `TaskRouter`, and configurable `HybridRouter`
- `RouterConfig` for routing parameters and weights
- Token-budget-aware context packing
- Context filtering helpers
- Public package exports and typed package marker
- Structured routing event objects for lightweight observability

## Production-readiness status

This release is suitable as a clean v0.1.0 reference implementation or starting point for agent systems. It keeps dependencies lightweight, uses standard library SQLite for durable storage, includes production-oriented docs, and validates tests, package builds, dependency audit, and CodeQL security analysis in CI.

## Known limitations

- The default semantic embedding fallback is deterministic and lightweight, not a substitute for production embeddings.
- Token estimation is approximate and should be replaced with model-specific tokenizers for strict budgets.
- SQLite is the only durable store included.
- Evaluation cases are still small and should be replaced with domain-specific benchmarks before production deployment.
- Prompt-injection filtering is not included. A lightweight PII/secret sanitization utility is available but is not automatically integrated into routing.
- Vector database adapters are not included yet.

## Next roadmap

- Consider optional vector database adapters based on issue-driven demand
- Add model-specific tokenizer integrations when strict token accounting is required
- Add richer real-world evaluation datasets
- Decide whether PII/secret sanitization should be opt-in at the `ContextPack` boundary
- Gather external user feedback before declaring a stable v1.0 API
