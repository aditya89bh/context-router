# context-router v0.1.0

## Summary

`context-router` v0.1.0 is a lightweight Python developer tool for routing the right context to the right agent at the right time. It demonstrates a production-oriented separation between memory storage, routing, token budgeting, packaging, and evaluation.

## Core features

- `ContextItem`, `ScoredContextItem`, and `ContextPack` data structures
- In-memory `MemoryStore`
- Standard-library `SQLiteMemoryStore` with context manager support and FTS-backed search when available
- `BaseRouter` contract
- `RecencyRouter`, `SemanticRouter`, `TaskRouter`, and configurable `HybridRouter`
- `RouterConfig` for routing parameters and weights
- Token-budget-aware context packing
- Context filtering helpers
- Public package exports and typed package marker

## Demos

## Production-readiness status

This release is suitable as a clean v0.1.0 reference implementation or starting point for agent systems. It keeps dependencies lightweight, uses standard library SQLite for durable storage, and includes CI across Python 3.10, 3.11, and 3.12.

## Known limitations

- The default semantic embedding fallback is deterministic and lightweight, not a substitute for production embeddings.
- Token estimation is approximate and should be replaced with model-specific tokenizers for strict budgets.
- SQLite is the only durable store included.
- Evaluation cases are small demo cases and should be replaced with domain-specific benchmarks.
- No prompt-injection, PII, or secret filtering is included yet.

## Next roadmap

- Add store protocol and vector database adapters
- Add model-specific tokenizer integrations
